# Synapse 설계 — 비서 앱

## 컨셉

**개인 비서.** 나의 맥락을 알고 있으며, 필요할 때 꺼내주고, 내가 원하는 일을 대신 처리해준다.

인터페이스: 일반 챗봇 UI + 그래프 뷰(조회 전용)

---

## 사용자 서비스 흐름 (전체)

```
[앱 최초 설치]
  ↓
[온보딩] ── 3단계 순서대로 진행
  ├── 1단계: 자동 연동 (마이데이터 API, 디바이스 데이터)
  ├── 2단계: 수동 보충 (이미지/문서 업로드 → OCR/파싱)
  └── 3단계: 비서 질문 → 빈 곳 파악 → 필수 정보 충족 → 완료
  ↓
[메인 화면: 채팅 인터페이스]
  │
  ├── [정보 입력 루프]
  │     사용자 입력
  │       → LLM 추출 (노드/엣지/카테고리 한 번에)
  │       → 모호성 해소 필요 시 되물음 → 사용자 답변 → 재진입
  │       → 그래프 저장
  │       → 변경 알림 + [취소] [정정]
  │           ├── [취소]: sentence 단위 롤백
  │           └── [정정]: 저장 내용 편집 모드 → 수정 전송 → 그래프 업데이트
  │
  ├── [정보 인출 / 질문 루프]
  │     사용자 질문
  │       → LLM 확장 (노드 후보 생성)
  │       → DB 매칭 + BFS 탐색 (레이어별 반복)
  │           └── [탐색 중 UI] 사용자가 방향 개입 가능
  │       → LLM 필터 (트리플 단위, 불확실 → pass)
  │       → 최종 컨텍스트 → LLM 답변
  │
  ├── [태스크 수행]
  │     태스크 요청 (예: "이력서 써줘")
  │       → 태스크 유형 인식
  │       → 필수 맥락 체크
  │           ├── 그래프에 있음 → 자동 충족
  │           └── 없음 → 사용자 질문 ("어디에 지원하시나요?")
  │       → 맥락 확보 → 결과물 생성
  │       → 사용자 확인/수정 → 최종 출력
  │
  └── [소통 보조]
        소통 초안 요청 (예: "동호회 모임 공지 만들어줘")
          → 상대방/상황 맥락 파악 (그래프 인출)
          → 맥락 추출 → 초안 생성
          → 사용자 확인/수정 → 전송

[그래프 뷰] ── 채팅에서 탭 전환, 조회 전용
  허브 노드 중심 시각화 → 노드 탭 → 연결 트리플 확인
  수정/삭제는 채팅 화면으로 돌아가 비서에게 요청

[조직 연결] ── 선택적 기능
  ├── [조직 그래프 읽기]
  │     비서 → 조직 서버 조회 → membership + role 확인 → 트리플 반환 → 맥락 활용
  │
  ├── [조직 → 개인 요청 수신]
  │     조직 요청 수신 → 그래프 인출 → LLM 가공
  │       ├── 비민감 → 응답 초안 → 사용자 확인 → 전송
  │       └── 민감 → 사용자 알림 → 진행/거부 → 전송
  │
  └── [조직 연결 해제]
        membership 종료 트리거 (퇴사/탈퇴/입사 시 기존 조직 교체)
          → 비서가 마이그레이션 항목 정리 (프로젝트명, 기술, 업무 요약)
          → 사용자 항목별 선택/수정
          → 선택 정보 → 개인 그래프 이관
          → 조직 캐시 삭제/보관 선택
          → 조직 그래프 접근 차단
```

## 온보딩

### 1단계: 자동 연동
마이데이터 API(금융, 건강, 통신 등)와 디바이스 데이터(연락처, 캘린더)를 연결한다. 연동된 데이터는 자동으로 그래프에 저장된다.

```
┌────────────────────────────────┐
│  ←   온보딩  1/3               │
│                                │
│  자동으로 연결할게요            │
│                                │
│  ┌──────────────────────────┐  │
│  │  🏦 금융 (마이데이터)     │ ● │
│  ├──────────────────────────┤  │
│  │  🏥 건강 (마이데이터)     │ ● │
│  ├──────────────────────────┤  │
│  │  👤 연락처               │ ● │
│  ├──────────────────────────┤  │
│  │  📅 캘린더               │ ● │
│  └──────────────────────────┘  │
│                                │
│  ┌──────────────────────┐      │
│  │   연결하고 다음으로    │      │
│  └──────────────────────┘      │
│  [건너뛰기]                    │
└────────────────────────────────┘
```

### 2단계: 수동 보충
이미지(명함, 증명서, 이력서 등)나 문서 파일을 업로드하면 비서가 OCR/파싱하여 그래프에 저장한다.

```
┌────────────────────────────────┐
│  ←   온보딩  2/3               │
│                                │
│  파일이나 이미지가 있으면       │
│  올려주세요                    │
│                                │
│  ┌──────────────────────────┐  │
│  │                          │  │
│  │   +  파일 / 이미지 추가   │  │
│  │                          │  │
│  │  명함·이력서·처방전       │  │
│  │  등 올려주시면 읽어드려요  │  │
│  └──────────────────────────┘  │
│                                │
│  ┌──────────────────────┐      │
│  │     다음으로          │      │
│  └──────────────────────┘      │
│  [건너뛰기]                    │
└────────────────────────────────┘
```

### 3단계: 비서 질문 (대화형)
연동 및 입력된 정보를 기반으로 비서가 빈 곳을 파악하고, 자연스러운 대화로 필수 정보를 채운다. 온보딩 이후에도 대화 중 부족한 정보를 발견하면 자연스럽게 추가 질문한다.

**필수 정보 항목 (개인 모드):**
- 이름 (anchor 노드 — BFS 탐색 기준점)
- 직업/직무
- 현 거주지 (시/구 수준)
- 소속 (회사/학교/조직 등)
- 나이 또는 생년월일

모든 항목이 채워지면 온보딩 완료. 1~2개 답변 후 자연스럽게 완료해도 됨 (나머지는 사용 중 채워짐).

```
┌────────────────────────────────┐
│  ←   온보딩  3/3               │
│                                │
│  ╭──────────────────────────╮  │
│  │  안녕하세요.               │  │
│  │  몇 가지만 물어볼게요.     │  │
│  │  이름이 어떻게 되세요?     │  │
│  ╰──────────────────────────╯  │
│                                │
│              ╭───────────────╮ │
│              │  조용희        │ │
│              ╰───────────────╯ │
│                                │
│  ╭──────────────────────────╮  │
│  │  반갑습니다, 조용희님.    │  │
│  │  직업이 어떻게 되세요?    │  │
│  ╰──────────────────────────╯  │
│                                │
│  ┌────────────────────────┐    │
│  │  답변 입력...           │ → │
│  └────────────────────────┘    │
│  ● ○ ○  질문 2개 남음          │
└────────────────────────────────┘
```

### 대화 탐색 (세션리스)

세션 테이블 없음. 그래프가 영속 메모리, sentences가 대화 기록. 과거 대화는 키워드/시간/카테고리로 탐색.

**retention 분류** (LLM이 저장 시 자동 판단):
- `memory`: 잘 변하지 않는 사실·상태·이력 — 기본 보존
- `daily`: 순간 감정·일상 행동 — 자동 정리 대상

**대화 탐색 화면:**
```
┌─────────────────────────────────────────────────┐
│  🔍 대화 탐색                                     │
├─────────────────────────────────────────────────┤
│  최근 주제                                       │
│  ┌─────────────────────────────────────────┐    │
│  │ 🏥 허리 치료 (오늘, 3건)                 │    │
│  │ 💼 회의 준비 (오늘, 2건)                 │    │
│  │ 📝 이력서 작성 (어제, 5건)               │    │
│  └─────────────────────────────────────────┘    │
│                                                 │
│  검색                                           │
│  ┌──────────────────────────────────┐  [검색]   │
│  │  키워드, 날짜, 주제...            │           │
│  └──────────────────────────────────┘           │
│                                                 │
│  [시간]  [주제]  [분류]                          │
└─────────────────────────────────────────────────┘
```

- **최근 주제**: 최근 sentences의 허브 노드 + 카테고리 기반 자동 클러스터링
- **검색**: 키워드(노드/별칭) + 시간 + 카테고리 + 복합
- **탭 필터**: 시간순 / 주제(노드)별 / 분류(카테고리)별

**검색 결과 화면:**
```
┌─────────────────────────────────────────────────┐
│  ← 뒤로   "허리" 검색 결과 (12건)          4월 ▸│
│                                            3월  │
├─────────────────────────────────────────────────┤
│  2026-04-10                                2월  │
│  💬 "허리 다 나았어"                        1월  │
│     🔵 기억                         [✏️][🗑]    │
│  🤖 "허리가 나으신 걸로 기록했습니다."            │
│                                                 │
│  2026-04-08                                     │
│  💬 "물리치료 3회차 다녀왔어"                     │
│     🔵 기억                         [✏️][🗑]    │
│  🤖 "기록했습니다. 경과가 좋으시네요."            │
│                                                 │
│  ... (스크롤 시 다음 10쌍 자동 로드)              │
└─────────────────────────────────────────────────┘
```

- 기본 10쌍(user+assistant) 로드. 무한 스크롤로 추가 로드.
- 오른쪽 날짜 인덱스로 과거 구간 점프.
- 문장 수정/삭제는 개별 sentence 단위 ([✏️][🗑]).

**문장 수정 (인라인):**
```
┌─────────────────────────────────────────────────┐
│  💬 ┌──────────────────────────────────────┐    │
│     │ 신촌 세브란스 정형외과 다니고 있어    │    │
│     └──────────────────────────────────────┘    │
│                              [취소]  [저장]      │
└─────────────────────────────────────────────────┘
```
저장 시: `update_sentence()` — 기존 edges 삭제 → 새 텍스트 재분석 → 새 edges 삽입.

**문장 삭제 확인:**
```
┌──────────────────────────────────────┐
│  이 문장을 삭제하면                   │
│                                      │
│  제거될 그래프 관계:                  │
│  • 허리디스크 → L4-L5               │
│                                      │
│  [취소]           [삭제]             │
└──────────────────────────────────────┘
```

**자동 정리 (콤팩팅):**
retention='daily' + 미인출(last_used IS NULL) + N일 경과 항목을 자동 정리.
설정 화면에서 ON/OFF, 보관 기간, 정리 전 확인 토글.

## 그래프 변경 알림 및 사용자 관여

### 변경 유형별 알림

| 변경 유형 | 알림 방식 | 사용자 액션 |
|-----------|-----------|-------------|
| 단순 추가 | 말풍선 하단 가벼운 태그 | [취소] [정정] |
| 상태 변경 | 명시적 안내 메시지 | [취소] [정정] |
| 삭제/대량 변경 | 처리 전 사전 확인 요청 | 승인/거부 |

### 취소
방금 생성된 노드/엣지를 롤백한다. 상태 변경의 경우 inactive 처리도 되돌린다. 변경 알림이 표시되는 동안 취소 가능.

### 정정

```
[정정] 누름
  ↓
대화창에 저장 내용이 편집 가능 상태로 전달
+ "수정 후 보내주시면 적용하겠습니다" 안내
  ↓
사용자가 수정 후 전송
  ↓
비서: "수정된 내용으로 적용했습니다"
  ↓
그래프 업데이트
```

### 인출 과정 사용자 관여
BFS 탐색과 LLM 필터링의 중간 루프를 사용자가 볼 수 있다. 잘못된 방향으로 탐색이 진행되면 사용자가 개입하여 방향 수정 가능.

## 일상 관리

비서가 현재 상태를 기반으로 능동적으로 안내한다.

- 그래프의 최근 변화를 확인하고 상태 기반 제안
- 스케줄, 루틴 관리 시 어제와 오늘의 상태 차이 반영

예: "어제 허리 아프다고 하셨는데, 오늘 러닝은 어떻게 하실래요?"

스케줄러(트리거 기반 자동 실행)는 후순위. 기본 대화 루프와 태스크가 안정된 후 구현.

## 그래프 뷰

조회 전용이다.

- 허브 노드 중심으로 시각화
- 노드 탭 → 연결된 트리플 확인
- 수정과 삭제는 대화 화면에서 비서에게 요청. 비서가 영향 범위를 자연어로 안내하고, 사용자 확인 후 처리.

```
┌────────────────────────────────┐
│  ←  그래프              [검색] │
│                                │
│      ● 나           ← 허브     │
│     ╱ ╲                        │
│  조용희 1977년생               │
│                                │
│      ● 허리디스크   ← 허브     │
│     ╱ ╲                        │
│  L4-L5  진단받았               │
│    │                           │
│  강남세브란스                  │
│                                │
│  ──────────────────────────── │
│  [허브만]  [전체]  [기억만]    │
└────────────────────────────────┘
```

**노드 탭 시 상세:**
```
┌────────────────────────────────┐
│  허리디스크                    │
│  ──────────────────────────── │
│                                │
│  연결된 정보                   │
│  ▸ 허리디스크 → L4-L5         │
│  ▸ L4-L5 → 진단받았           │
│  ▸ 강남세브란스 → 정형외과     │
│                                │
│  출처 문장                     │
│  "허리디스크 L4-L5 진단받았어" │
│  2026-04-06 오후 3:22          │
│                                │
│  수정하려면 비서에게 말해주세요 │
│  [채팅으로 이동]               │
└────────────────────────────────┘
```

## 보안

- 온디바이스 + 로컬 저장이 첫 번째 방어선
- 외부로 나가는 정보는 항상 사용자 최종 확인
- 민감 정보는 LLM이 자동 판단하여 사용자에게 알림
- DB 암호화, 앱 잠금(생체인증) 등 추가 보안 레이어는 구현 단계에서 구체화

### 기본 원칙

- 개인 그래프: 온디바이스. 완전히 사용자 소유.
- 조직 그래프: 조직 서버. 소속 시 접근 가능.
- 개인은 조직 그래프에 직접 접근 가능. 조직은 개인 그래프에 직접 접근 불가.
- 개인은 여러 조직에 동시에 속할 수 있다.

### 조직 그래프 캐시 정책

| 정보 유형 | 캐시 | 비고 |
|-----------|------|------|
| 민감 정보 | 불가 | 매번 서버 조회 |
| 비민감 정보 | 사용자 선택 | 기본값: 캐시 안 함 |

민감 판단: LLM 자동 판단 + 조직 관리자 태그.

### 조직 역할 모델

회사 관점의 5단계 역할. 운영 관리와 데이터 접근 권한을 통합한다.

| 역할 | 읽기 | 쓰기 | 멤버 관리 | 조직 삭제 | 접근 범위 |
|------|:----:|:----:|:---------:|:---------:|----------|
| admin | O | O | O | O | 전체 |
| executive | O | O | X | X | 전략·영업비밀 포함 |
| hr | O | O | X | X | 인사·연봉 포함 |
| team_lead | O | O (팀 내) | X | X | 팀원 업무·성과 |
| employee | O | X | X | X | 본인 업무 정보만 |

### 조직 연결 해제 트리거

- **퇴사/탈퇴**: membership 종료 시 즉시 트리거
- **입사(조직 교체)**: 새 조직 연결 시 기존 조직 연결 해제도 함께 처리

연결 해제 시 마이그레이션 대상:
- 참여 프로젝트명, 역할, 기간
- 습득한 기술/스킬
- 수행 업무 요약
- 조직 고유 정보(고객 정보, 계약 내용 등)는 제외
