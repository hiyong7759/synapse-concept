알려진 사실 목록에서 현재 입력으로 인해 상태가 바뀐 문장을 분류하라.
JSON만 출력. 다른 텍스트 금지.

출력 형식:
{"deactivate":[sentence_id, ...], "pending":[sentence_id, ...]}

입력 구조:
- 현재 입력(원문 + Kiwi 명사 + Kiwi 용언 lemma)
- 알려진 사실 목록 — 각 항목은 [sentence_id] 원문 + 명사 + 용언

판정 규칙:
- **명사(주체) 공유 + 용언 대립** → deactivate
  예: 현재 "허리 낫" vs 사실 "허리 아프" — 공통 주체 '허리' + 반대 상태.
- **명사 공유 but 용언 관계 애매** → pending (사용자 확인 필요)
  예: 현재 "운동 쉬" vs 사실 "운동 매일" — 일시적인지 중단인지 불명.
- **명사 공유 없음** 또는 **용언 동일·보완** → 어느 쪽도 아님
  예: 현재 "병원 예약" vs 사실 "병원 다녀오" — 같은 병원이지만 무효화 관계 아님.
- **부정부사(안/못) 추가/제거** → deactivate
  예: 현재 "운동 안 하" vs 사실 "운동 하" — MAG 부정 추가 = 반대.

엄격성:
- 확실한 무효만 deactivate, 정말 애매한 것만 pending. 과잉 분류 금지.
- 주체(명사) 교집합 0 이면 항상 제외.
- 둘 다 없으면 {"deactivate":[], "pending":[]}.

예시:
현재 입력:
  원문: 허리 나았어
  명사: [허리]
  용언: [낫]
알려진 사실:
  [1] 원문: 허리 아파서 병원 다녀왔어
      명사: [허리, 병원]
      용언: [아프, 다녀오]
  [2] 원문: 나 아침에 커피 마셔
      명사: [나, 아침, 커피]
      용언: [마시]

출력: {"deactivate":[1], "pending":[]}
  (이유: [1]은 '허리' 공유 + '아프 → 낫' 대립 / [2]는 명사 교집합 0)
