## 1. 배경 — 이번 세션에서 완료된 것

### 핵심 아키텍처 변경 (구현 완료)

대화 세션과 문장을 그래프에 연결하는 구조로 전면 개편.

**변경된 파일:**
- `engine/db.py` — `sessions`, `sentences` 테이블 추가, `edges.sentence_id` FK
- `engine/morpheme.py` — `analyze_sentences()` 추가 (Stanza 문장 경계 활용)
- `engine/save.py` — 문단→문장 단위 처리, sentence_id 연결, session_id 파라미터
- `engine/retrieve.py` — sentence dedup 기반 LLM 필터링
- `engine/llm.py` — `SYSTEM_RETRIEVE_FILTER` 문장 기반으로 변경

**새 DB 스키마 (v7):**
```sql
sessions:  id, created_at
sentences: id, session_id, paragraph_index, text, created_at
nodes:     id, name, status, created_at, updated_at
edges:     id, source_node_id, target_node_id, label, sentence_id, created_at, last_used
aliases:   alias TEXT PRIMARY KEY, node_id
```

**저장 파이프라인:**
```
입력 텍스트
  → \n\n 기준 문단 분리
  → analyze_sentences(): Stanza 문장 경계 분리
  → sentences INSERT (session_id, paragraph_index, text)
  → 형태소 분석 → 트리플
  → LLM 전처리 (상태변경/치환/모호성)
  → edges INSERT (sentence_id 포함)
```

**인출 파이프라인:**
```
질문
  → LLM 키워드 확장
  → BFS: _get_triples() LEFT JOIN sentences
  → sentence_id dedup → LLM 문장 단위 필터
  → 최종 답변
```

## 2. 검증 결과 (gemma4:e2b 기준)

### Retention 분류 — 83% 정확

| 분류 | 케이스 | 결과 |
|------|--------|------|
| ✓ 정확 | 날씨, 커피, 피곤해, 파스타 → 일상 | OK |
| ✓ 정확 | 출생연도, 허리디스크, 맥미니, 직업 → 기억 | OK |
| ✓ 정확 | 처방 → 기억 | OK |
| △ 오분류 | "요즘 운동 시작했어" → 일상 (기억이어야 함) | 프롬프트 보정 필요 |
| △ 오분류 | "오늘 이직 첫날이야" → 일상 (기억이어야 함) | "오늘+이벤트" 패턴 |

**보정 규칙 추가 필요**: "오늘"이 포함되어도 이직/입사/진단/이사 같은 이벤트는 permanent.

### 상태변경 감지 — 67% 정확

| 케이스 | 결과 |
|--------|------|
| 명시적 정정 ("잘못 말했어, X가 아니라 Y") | ✓ state_change 감지 |
| 상태변화 ("이직했어") | ✓ state_change 감지 |
| 나이 → 출생연도 치환 | ✓ 잘 처리 |
| 기존 없을 때 state_change 미생성 | ✓ 정상 |
| 단순 재입력 ("나 1977년생이야" + 기존 1978년) | ✗ state_change 미감지 |
| 모호성 ("거기 그만뒀어") | ✗ 질문 안 하고 임의 선택 |

**결론**: 단순 재입력 정정은 대화로 불안정 → 세션 편집 UI가 필수 보완책.

## 3. 설계 확정 사항

### 3-1. 저장 전략
- **모든 입력 전부 저장** (선택 저장 없음, 마찰 제거)
- LLM이 자동으로 저장 세션(진술)과 인출 세션(질문) 구분
  - 의문형 어미로 끝나는 입력 → 저장 없이 인출만
  - 사실/상태 진술 → 저장

### 3-2. Retention 분류
- `기억` (permanent): 잘 변하지 않는 사실, 상태, 관계, 이력
- `일상` (ephemeral): 순간적 활동, 일상 감정, 오늘의 일
- gemma4가 `SYSTEM_SAVE` 응답에 `retention` 필드로 반환
- `sentences.retention` 컬럼에 저장

### 3-3. 정정 처리
- 대화 중 자동 감지 (불안정, 보조 수단)
- **주요 수단: 세션 편집 UI에서 원문 직접 수정**
- 수정 시: 기존 sentence 텍스트 업데이트 → 연결 edges 삭제 → 새 텍스트 재분석 → 새 edges 삽입
- 나이는 `SYSTEM_SAVE`에서 출생연도로 자동 치환 (2026 기준)

### 3-4. 세션 삭제
- 삭제 시 sentences + 연결 edges + 고아 노드 전부 cascade 삭제
- 삭제 전 "제거될 항목 미리보기" 표시
- **예외 없음**: 단순하게 전부 삭제

### 3-5. 콤팩팅
- 세션 내 `기억` 항목만 자동 선택 → 보관함으로 이동
- `일상` 항목은 기본 숨김 (오분류 대응용 "펼치기" 제공)
- 콤팩팅 후 원본 세션 삭제

### 3-6. 장기 보관함
- `sessions.type`: `conversation` | `archive`
- archive 세션: 월별 자동 생성 ("2026-04 보관함"), 삭제 불가
- 콤팩팅으로만 항목 유입, 개별 수정/삭제는 가능

### 3-7. 패턴 추출 (연기)
- 일상 데이터 누적 → 반복 패턴 → permanent 지식 승격
- sentences 테이블에 원문 전부 있으므로 나중에 `consolidate.py`로 구현 가능
- 이번 구현 범위 외
