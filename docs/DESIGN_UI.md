# Synapse 설계 — UI/UX 와이어프레임

## 디자인 방향

### 컨셉
"나만 아는 비서." 외부에 데이터가 없다는 사실이 디자인에서도 느껴져야 한다.
따뜻하고 정밀하며, 기계적이지 않은 인터페이스.

### 토큰

| 항목 | 값 |
|------|----|
| Background | `#0A0B0E` (딥 네이비블랙) |
| Surface | `#141519` (다크 카드) |
| Border | `#252730` |
| Text Primary | `#E8E4D9` (웜 크림) |
| Text Secondary | `#7A7B82` |
| Accent | `#C8A96E` (앰버골드 — 하이퍼그래프 노드, 핵심 액션) |
| Success | `#5DAB8A` |
| Danger | `#C85D5D` |
| Hyperedge Line | `#3A3D4A` (하이퍼엣지 시각화 선) |

### 타이포그래피

| 역할 | 폰트 | 용도 |
|------|------|------|
| Display | Playfair Display | 로고, 온보딩 헤드라인 |
| Body KR | Noto Sans KR | 대화, 본문 전반 |
| Mono | JetBrains Mono | 하이퍼그래프 노드 이름, 트리플 표시 |

### 원칙

채팅·하이퍼그래프 뷰 동작 4개 원칙은 **`docs/DESIGN_PRINCIPLES.md §5` UI 원칙** 참고.
(변경 알림 비방해 · 탐색 상태 노출 · `/hypergraph` 별도 라우트 · 조직 연결 인디케이터)

탐색 상태 디자인 규칙:
- 미니 하이퍼그래프는 실시간 노드 확장을 점 애니메이션으로 표현.
- 진입한 노드: 앰버 채움 ●. 탐색 예정: 빈 원 ○. 하이퍼엣지(문장·카테고리 바구니)는 가는 선으로 노드들을 묶어 표시.

## Screen 5 — 하이퍼그래프 뷰 (`/hypergraph`)

별도 라우트. vis-network 기반 인터랙티브 시각화. 조회 전용. 하이퍼엣지(문장·카테고리 바구니)는 노드들을 감싸는 반투명 영역 또는 여러 노드를 동시에 잇는 선으로 표현.

```
┌──────────┬──────────────────────────────────┐
│ Synapse  │                                  │
│ ← 채팅   │      ●  허리       ← 허브(앰버)  │
│          │     ╱ ╲                          │
│ ⌕ 검색   │   아프  병원                     │
│          │     │     ╲                      │
│ 노드 42  │    었      물리치료              │
│ 바구니 87│                                  │
│          │           ●  직장  ← 허브        │
│ [전체]   │          ╱ ╲                     │
│ [허브만] │        삼성  재직                │
│          │                                  │
│ 깊이     │                [+] [fit] [-]     │
│ [0──●─5] │                                  │
└──────────┴──────────────────────────────────┘
```

- **좌측 사이드바**: 검색 + [전체/허브만] 토글 + BFS 깊이 슬라이더(0~5) + 노드/바구니(문장+카테고리) 통계. 접기 가능.
- **모바일**: 사이드바 숨김, FAB 버튼(☰)으로 접근.
- **호버 디밍**: Obsidian 스타일 — 선택 노드+같은 바구니 멤버만 하이라이트, 나머지 흐리게.
- **줌 컨트롤**: [+] [fit] [-] 우측 하단.

노드 탭 시 (우측 상세 패널):
```
┌────────────────────────────────┐
│  허리                    [✕]  │
│  HUB                          │
│  ──────────────────────────── │
│                                │
│  → 아프                        │  ← 클릭 시 해당 노드로 포커스
│  → L4-L5                       │
│  ← 병원                        │
│  → 물리치료                     │
│                                │
│  연결 4개 (→3 ←1) · 허브 노드  │
│                                │
│  수정하려면 비서에게 말해주세요  │
│  [채팅으로 이동]                │
└────────────────────────────────┘
```

### 7-1. 모드 분리 진입점

Screen 2의 채팅 입력창에서 **긴 글 아이콘** 또는 단축키(⌘⇧N)로 진입. 모드는 여기서 **한 번 결정되면 작성 중 전환 불가**.

```
┌────────────────────────────────┐
│  SYNAPSE — 새 기록              │
│                                │
│  어떤 형태로 저장할까요?        │
│                                │
│  ╭──────────────────────────╮  │
│  │  💬  짧은 메모·대화       │  │  ← chat 모드
│  │      한 줄 감상, 즉흥 기록 │  │
│  ╰──────────────────────────╯  │
│                                │
│  ╭──────────────────────────╮  │
│  │  📝  구조화된 글           │  │  ← markdown 모드 (이 화면)
│  │      회의록·규정·레시피    │  │
│  ╰──────────────────────────╯  │
│                                │
│  [취소]                        │
└────────────────────────────────┘
```

진입점 디자인 규칙
- 두 카드 균등 배치. 선택 시 앰버 테두리 하이라이트.
- 각 카드 아래에 **예시 문구 한 줄** (용도 환기).
- "markdown 모드" 같은 기술용어 대신 "구조화된 글" 워딩 사용.

### 7-2. 에디터 기본 레이아웃

```
┌──────────────────────────────────────────────────┐
│  ← 새 기록           [양식▾] [분류▾]       [저장] │  ← 상단 바
├──────────────────────────────────┬───────────────┤
│                                  │               │
│  # 취업규칙                       │  📖 작성 참고 │  ← 우측 참조 패널
│                                  │  ─────────────│     (접기 가능)
│  - 회사:: 주식회사 더나은          │  제목         │
│  - 개정일:: 2025-04-30            │  # 이상 다단계│
│                                  │               │
│  ## 제1장 총칙                    │  용어 정의    │
│                                  │  - 이름:: 뜻  │
│  ### 제1조 (목적)                 │               │
│                                  │  나열         │
│  이 규칙은 ...                    │  - 항목       │
│                                  │               │
│                                  │  한 줄 분류   │
│                                  │  # A.B.C      │
│                                  │               │
│                                  │  [자세히 보기]│
├──────────────────────────────────┴───────────────┤
│  12:34   ⓘ 분류 3단계 · 문단 5개        [ ⌨ / ] │  ← 상태 바
└──────────────────────────────────────────────────┘
```

레이아웃 규칙
- 좌측 에디터 70% / 우측 참조 패널 30%. 패널은 상단 바 `⊟` 로 접을 수 있음.
- 상단 바 `[양식▾]` — INPUT_GUIDE §4 문서 유형별 템플릿 삽입 드롭다운 (법령·회의록·레시피·사양서·일지).
- 상단 바 `[분류▾]` — 기존 분류 둘러보기·경로 삽입.
- 하단 상태 바 — 현재 저장될 분류 단계 수·문단 수 실시간 표시. 우측 `⌨ /` 는 슬래시 커맨드 힌트.
- 참조 패널은 **읽기 전용 요약**. 각 줄 클릭 시 인라인 예시 확장.

### 7-3. 슬래시 커맨드 (`/`)

줄 시작에서 `/` 입력 시 팝업. 커서 위치에 오버레이.

```
│  ### 제1조 (목적)                                │
│  /| (커서)                                       │
│  ┌──────────────────────────────────────────┐    │
│  │  /양식       문서 유형별 템플릿 삽입      │    │
│  │  /분류       내가 만든 분류 둘러보기      │    │
│  │  /속성       "이름:: 값" 빈칸 넣기        │    │
│  │  /목록       여러 줄 나열 빈칸 넣기       │    │
│  │  /오늘       오늘 날짜 자동 입력          │    │
│  │  /도움말     작성 가이드 열기             │    │
│  └──────────────────────────────────────────┘    │
```

커맨드 규칙
- 줄 시작 `/` 에서만 트리거. 본문 중간 `/` 는 일반 문자.
- ↑↓ 로 이동, Enter 로 선택, Esc 로 닫기.
- 한국어 키워드 먼저. 영문 별칭은 보조 (내부 구현에서 alias 매핑).
- 커맨드명은 **개발 용어 배제** — "template" 대신 "양식", "category" 대신 "분류".

`/분류` 선택 시 카테고리 탐색기 전개:

```
│  ┌──────────────────────────────────────────┐    │
│  │  분류 둘러보기                            │    │
│  │  ─────────────────────────────────────    │    │
│  │  📁 취업규칙                              │    │
│  │    📁 제1장 총칙 (3)                     │    │
│  │    📁 제2장 채용 및 근로계약 (3)          │    │
│  │    📁 제4장 인사 ▶                        │    │
│  │       └ 📁 제1절 인사위원회 (3)          │    │
│  │       └ 📁 제2절 배치·전직·승진 ▶        │    │
│  │          └ 📁 제16조 인사발령 원칙 (9)    │    │
│  │  📁 건강                                  │    │
│  │  ─────────────────────────────────────    │    │
│  │  [+ 새 분류 만들기]                       │    │
│  └──────────────────────────────────────────┘    │
```

탐색기 규칙
- 트리 확장 단계 제한 없음 — 사용자 분류 깊이만큼 표시.
- 괄호 숫자 `(N)` — 해당 분류에 담긴 문장 수. 활성도 가늠.
- 경로 선택 시 `# 취업규칙.제1장 총칙.제1조 (목적)` 형태로 **한 줄 삽입**.
- "+ 새 분류 만들기" — 빈 heading 템플릿 삽입.

### 7-4. 실시간 린터 (자주 하는 실수 안내)

`INPUT_GUIDE.md §5`의 6가지 실수를 **빨간 밑줄 + 호버 툴팁**으로 표시. LLM 호출 없이 정규식·파싱만으로 판단.

```
│  - 보직: 직원의 자질에 따라 특정 직무를 명함.   │
│    ~~~~~~                                      │
│    ┌─────────────────────────────────────┐     │
│    │ 이 문장을 어느 쪽으로 저장할까요?    │     │
│    │                                      │     │
│    │ 🔵 "보직" 단어의 뜻 설명             │     │
│    │    나중에 "보직이 뭐야?" 물으면       │     │
│    │    이 문장이 바로 나와요             │     │
│    │                                      │     │
│    │ ⚪ 그냥 평범한 한 줄                 │     │
│    │    다른 문장들 사이에 섞여 저장돼요   │     │
│    │                                      │     │
│    │ [단어 설명이에요] [일반 문장이에요]  │     │
│    └─────────────────────────────────────┘     │
```

린터 규칙
- **의도 확인형** — "틀렸어요" 대신 "이렇게 이해했는데 맞나요?" 질문 형식.
- 기호 이름("콜론", "하이픈") · 줄 번호("2번째 줄") 등 개발 용어 금지. 문제 부분을 **그대로 화면에** 표시.
- 버튼 선택 시 자동 수정 (예: `:` → `::`).
- 적용 대상 6가지 실수 — 콜론 하나 / 조항 제목 자유 문장 / 제목·본문 한 줄 섞음 / 분류 이름에 `.` / 연속 점 / `- key::` 값 비어있음.

### 7-5. 저장 전 확인 모달

상단 바 `[저장]` 클릭 → 자동 점검 (0.1초 내) → 자동 감지 가능한 항목은 먼저 해결 제안, 사람 판단이 필요한 항목은 체크박스로.

```
┌────────────────────────────────────────────────┐
│  저장 전 확인                                   │
│  ────────────────────────────────────────       │
│                                                 │
│  ⚠️ 자동으로 찾은 부분                          │
│    (린터가 경고 중인 줄이 3개 있어요.           │
│     먼저 확인하시겠어요?)                       │
│    [ 경고 줄로 가기 ]                           │
│                                                 │
│  ────────────────────────────────────────       │
│                                                 │
│  ✏️ 직접 확인해 주세요                           │
│    ☐ 모드가 맞는지 (구조화된 글이 맞는지)       │
│    ☐ 제목들이 분류로서 의미 있는지              │
│                                                 │
│  ────────────────────────────────────────       │
│                                                 │
│  이대로 저장하면                                │
│   · 분류 경로: 취업규칙 > 제1장 총칙 > 제1조    │
│   · 새 분류 3개 · 새 문장 5개                   │
│                                                 │
│  [ 조금 더 고칠게요 ]   [ 이대로 저장 ]         │
└────────────────────────────────────────────────┘
```

모달 규칙
- **경고 > 체크박스 > 저장 요약** 3단 구조.
- 체크박스는 **선택 사항** (강제 체크 아님). 저장은 언제든 가능.
- 저장 요약 — 이 저장이 만들어낼 "새 분류 N개 · 새 문장 M개" 표시. 사용자가 저장 범위를 감각적으로 파악.
- 이전 세션 설정에 따라 "다음부터 이 모달 건너뛰기" 옵션 제공 (숙련자용).

### 7-6. 키보드 인터랙션

| 키 | 동작 |
|---|---|
| `/` (줄 시작) | 슬래시 커맨드 열기 |
| `⌘K` | 슬래시 커맨드 열기 (줄 중간에서도) |
| `⌘S` | 저장 (확인 모달) |
| `⌘⏎` | 저장 (모달 건너뛰기) |
| `⌘P` | 분류 둘러보기 바로 열기 |
| `⌘⇧N` | 새 기록 (채팅에서 진입) |
| `Esc` | 팝업 닫기 / 에디터 나가기 |

## 컴포넌트 정의

### 비서 말풍선

```
╭─────────────────────╮
│ (비서 아이콘) 텍스트  │
│                     │
╰─────────────────────╯
HH:MM AM
```
- 배경: `#1C1E24`
- 좌측 정렬, 아이콘 없음 (텍스트만)
- 비서 이름 레이블 없음 (단일 비서)

### 사용자 말풍선

```
              ╭─────────────────╮
              │           텍스트 │
              ╰─────────────────╯
                          HH:MM AM
```
- 배경: `#1E2030` (약간 블루 틴트)
- 우측 정렬

### 변경 알림 (단순)

```
┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
▸ source — label — target
▸ source — target
[취소]  [정정]
┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
```

### 변경 알림 (상태 변경)

```
╔═══════════════════════╗
║ ⚡ 상태 변경           ║
║ node → node  [비활성] ║
║ [취소]  [정정]        ║
╚═══════════════════════╝
```

### 결과물 카드

```
╔═══════════════════════╗  ← 앰버 좌측 보더
║ 📄 제목                ║
║ ─────────────────────  ║
║ 미리보기 텍스트...     ║
║                        ║
║ [전문 보기]  [수정 요청]║
╚═══════════════════════╝
```
