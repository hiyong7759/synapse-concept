## 라우트 (모바일 우선·데스크톱 통합)

같은 Flutter 위젯이 모바일·데스크톱에서 자연 확장. post 사이드바는 모바일에선 슬라이드 패널(`☰`), 데스크톱에선 고정 패널로 노출.

- `/note` — 지식 축적 (자동저장 + 의미 처리) + **해당 노트 그래프 패널**
- `/synapse` — 인출·융합 세션 (통찰 승격) + **해당 시냅스 세션 그래프 패널**
- `/hypergraph` — **전체 누적 그래프** (조회 전용)

`/review` (검수 전용) 는 후속 PLAN.

### 그래프 패널 vs /hypergraph 라우트

| 위치 | 범위 | 용도 |
|---|---|---|
| `/note` 그래프 패널 | 지금 보고 있는 post 의 노드/하이퍼엣지만 | 입력 → 의미 처리 직후 "내가 친 게 그래프에 어떻게 들어갔지" 즉시 검증 |
| `/synapse` 그래프 패널 | 현재 시냅스 세션의 retrieve 캐시 노드만 | 답변이 어떤 노드에서 끌어왔는지 추적 |
| `/hypergraph` 라우트 | 전체 누적 그래프 (모든 post 통합) | 누적 지식의 허브·도메인 부상 확인. 통찰 승격 후 허브 변화 검증 |

## 사용자 서비스 흐름

```
[앱 최초 설치]
  ↓
[기본 화면] ── 좌측 post 사이드바 (번호·제목·수정시간 + ✦ insight) + 우측 라우트 컨텐츠
  │
  ├── /note ── 지식 축적 (모든 입력의 단일 그릇)
  │     │
  │     ├── 입력 (한 줄 메모 / 긴 마크다운 / 평문 — 형식 인지·선택 없음)
  │     │     ↓
  │     ├── [자동저장] (1.5초 디바운스 + 페이지 이탈, LLM 미사용)
  │     │     posts.source UPDATE 한 줄. <50ms. 손실 방지 단일 목적.
  │     │     상태 표시: "변경됨 / 저장 중… / ✓ 저장됨 (방금)"
  │     │     ↓
  │     ├── [의미 처리] (사용자 명시 ⌘S 또는 "정리" 버튼)
  │     │     상태 표시: "⚙ 정리 중… → ✦ 정리됨"
  │     │     posts.source flush → sentences 재계산 → Kiwi 노드 추출
  │     │     → node_sentence_mentions / sentence_categories / node_category_mentions
  │     │     → LLM 정정 후보 생성 (별칭 보호 + 자모 거리 사전 필터)
  │     │     ↓
  │     ├── [정정 카드 인라인]  자동 적용 금지 (원칙 14-③)
  │     │     "스ㅏ타벅스 → 스타벅스?" [적용] [무시]
  │     │     [적용] 시: alias 등록 + source/sentences 갱신
  │     │     별칭 등록 토큰(`스벅` 등)은 카드에 안 뜸 (사용자 정감 보존)
  │     │     ↓
  │     └── 재진입·이어쓰기·post 삭제 (사이드바 호버 액션)
  │
  └── /synapse ── 인출·융합 세션 (지식 발화)
        │
        ├── 질문 입력
        │     ↓
        ├── [retrieve-expand 어댑터] 노드 후보 키워드 확장
        │     ↓
        ├── [Kiwi 보강] 형태소 분석으로 후보 추가
        │     ↓
        ├── [BFS 인출] 세 경로 (문장 바구니 + 카테고리 공유 + 사용자 heading 계층)
        │     ↓
        ├── [retrieve-filter] 무관 sentence 제거 (불확실 → pass)
        │     ↓
        ├── [synapse-answer] 컨텍스트 sentences 합성 (원문 나열 아닌 융합)
        │     ↓
        ├── 답변 카드 표시 + 메시지 hover 시 [⬆ 통찰로 승격]
        │     ↓
        └── [통찰 승격] (사용자 명시 클릭)
              새 posts(kind='insight', title=본문 첫 행)
              + 본체 sentence(origin='insight', 편집 불가)
              + 시냅스 세션 retrieve 캐시 노드 전부와 node_sentence_mentions 일괄 INSERT
                (Hebbian — 함께 발화한 것이 모두 엮임)
              → /note 사이드바에 ✦ 아이콘 + 앰버 보더로 표시
              → 다음 retrieve 에서 우선 점화 (다음 시냅스의 중력)
```

**핵심**:
- 모드 선택 없음 — 사용자는 입력 형식을 인지·선택하지 않음. 평문이든 마크다운이든 같은 그릇 (`note` post).
- 자동저장과 의미 처리의 두 층 분리 — 매 입력 LLM 호출 폐지로 모바일 배터리·발열 보호 (원칙 13).
- 정정은 제안만 — 사용자 [적용] 클릭 강제 (원칙 14-③). 자동 표준화로 사용자 정감 변질 금지.
- 통찰 = 함께 발화한 것의 허브 — 본체 1 sentence 가 retrieve 캐시 노드 전부와 자동 일괄 연결 (Hebbian).
- 시간 기반 자동 경계 없음 — post 는 사용자가 명시적으로 닫을 때까지 열려있고 재진입 가능.

## /note — 지식 축적

### 데스크톱 (3-pane: post 사이드바 + 본문 + 그래프 패널)

```
┌──────────────────────────────────────────────────────────────────────────────┐
│ SYNAPSE                                       /note    /synapse   /hypergraph │ ← 상단 바
├──────────────┬──────────────────────────────────────┬────────────────────────┤
│ ＋ 새 노트    │  ✓ 저장됨 (방금)         ⌘S 정리   │  📊 이 노트 그래프  ⊟  │
│              │  ──────────────────────────────────  │  ──────────────────── │
│ ── 노트 ──   │                                      │   (node-centric, 옵션 ε)│
│ 12 허리 치료 │  # 더나은                            │                        │
│    4분 전    │  ## 개발팀                           │       ●박지수           │
│ ──────────── │                                      │           │             │
│ 11 팀 회의록 │  - 팀장:: 박지수                     │       ●개발팀 (hub)      │
│    2시간 전  │  - 프론트엔드 김민수                  │       ╱  │  ╲          │
│ ──────────── │                                      │   ●더나은 │  ●김민수    │
│ 10 ✦ 모바일  │  오늘 회의가 세 개 연속이었다.        │           │             │
│    우선 결정 │  김민수가 새 컴포넌트 시안 잡아왔다.   │       ●회의             │
│    어제      │                                      │           │             │
│              │                                      │       ●컴포넌트          │
│ ── 시냅스 ── │                                      │           │             │
│ 9 허리 치료  │                                      │       ●시안             │
│    3시간 전  │                                      │                        │
│              │                                      │  ─── (선 = 공출현)     │
│              │                                      │  카테고리: 더나운/개발팀 │
│              │                                      │  노드 7  바구니 4       │
└──────────────┴──────────────────────────────────────┴────────────────────────┘
```

- 우측 그래프 패널은 현재 보고 있는 post 의 노드/하이퍼엣지만 시각화 (`/hypergraph` 의 `GraphView` 코어를 `postId` 필터로 wrap)
- 의미 처리 직후 새 노드가 즉시 패널에 반영
- `⊟` 로 패널 접기 가능 (본문 풀폭으로 확장)
- **노드 클릭** 시 같은 sentence 멤버 하이라이트 + 우측 detail panel (모바일은 하단 시트) — hover 갈음, 모바일·데스크톱 통일
- 노드 크기 = mention degree, 색상 = 시드 카테고리, `✦` insight 노드 = 앰버 + 가장 큼 + 외곽 링

- `☰` 탭 → post 사이드바 슬라이드
- `📊` 탭 → 그래프 패널 풀스크린 또는 하단 시트로 전환

### post 사이드바

- 번호 + 제목 + 수정시간
- `kind='insight'` post 는 `✦` 아이콘 + 앰버 보더
- `kind='note'` / `kind='synapse'` 섹션으로 그룹화
- 호버 시 [삭제] 액션 (확인 모달 후 sentences·mentions CASCADE)
- 클릭 시 source 그대로 로드 (재진입·이어쓰기)

### 자동저장 상태 + 의미 처리 트리거

상단 바에 짧은 상태 줄로 노출. 두 동작이 **시각적으로 구분**:

| 상태 | 표시 | 트리거 |
|---|---|---|
| 자동저장 | `변경됨` → `저장 중…` → `✓ 저장됨 (방금)` | 1.5초 디바운스 + 페이지 이탈 |
| 의미 처리 | `⌘S 정리` 버튼 → `⚙ 정리 중…` → `✦ 정리됨` | 사용자 명시 (⌘S 또는 버튼) |

자동저장은 항상 백그라운드, 의미 처리는 사용자가 의도를 줄 때만.

### LLM 정정 카드 인라인

의미 처리 직후 / 재진입 시 본문 끝에 카드로 노출. 자동 적용 금지.

```
─── LLM 정정 후보 ───
┌──────────────────────────────────────┐
│  스ㅏ타벅스 → 스타벅스              │
│  근거: 자모 거리 1, 별칭 미등록       │
│  [적용]  [무시]                      │
└──────────────────────────────────────┘
┌──────────────────────────────────────┐
│  김민수 → 김민슈                     │
│  근거: 자모 거리 1                    │
│  [적용]  [무시]                      │
└──────────────────────────────────────┘
```

- 별칭 등록 토큰 (`스벅` 등) 은 카드에 안 뜸 (사용자 정감 보존).
- [적용] 클릭 시: aliases INSERT (origin='user') + posts.source / sentences 갱신.
- [무시] 클릭 시: 카드만 닫힘. 같은 토큰 캐싱되어 다음 의미 처리 때 다시 안 뜸.

### 본문 직접 편집

`/note` 본문이 항상 편집 가능 상태이므로 별도 "정정" 액션 불필요. 본문 직접 수정은 입력 영역에서 즉시 가능 (자동저장이 백그라운드로 받음).

### BFS 탐색 중 시각화

답변 생성 전 단계가 사용자에게 보임. 잘못된 방향으로 탐색이 진행되면 사용자 개입 가능.

```
◌ 탐색 중...

  허리 ●─── 디스크 ●─── L4-L5 ●
       ╲                       ╲
        병원 ○                  진단 ○
        ╲                       ╲
         강남세브란스 ○          물리치료 ○

  ● 진입한 노드   ○ 탐색 예정 노드
  [이 방향 제외하기]
```

복잡해지면 허브 노드 2~3개만 표시하고 "...외 N개 노드" 처리.

### 답변 카드 + 통찰 승격

```
╭──────────────────────────────────────────╮
│ A: ...답변 본문 (마크다운 렌더)...        │
│                                          │
│  [⬆ 통찰로 승격]   [재질문]   [복사]     │
╰──────────────────────────────────────────╯
```

`[⬆ 통찰로 승격]` 클릭 → 확인 모달:

```
┌──────────────────────────────────────┐
│  통찰로 승격하시겠어요?                │
│                                      │
│  새 ✦ 통찰 post 가 생성되며,           │
│  이 시냅스 세션에서 함께 발화한        │
│  노드 12 개가 자동 연결됩니다.         │
│                                      │
│  [취소]                  [승격]      │
└──────────────────────────────────────┘
```

승격 후:
- `/note` 사이드바에 `✦` 아이콘 + 앰버 보더로 새 post 표시
- 본체 sentence 는 편집 불가 (시도 시 토스트 "통찰은 편집할 수 없어요. 삭제는 /review 에서.")
- 다음 retrieve 에서 우선 점화 (다음 시냅스의 중력)

## 변경 알림 및 사용자 관여

### 변경 유형별 알림

| 변경 유형 | 알림 방식 | 사용자 액션 |
|-----------|-----------|-------------|
| 지식 축적 (`kind='note'`) | 의미 처리 응답에 `correction_candidates` 누적, 인라인 카드 표시 | [적용] [무시] |
| 통찰 승격 (`kind='insight'`) | 시냅스 세션에서 `[⬆ 통찰로 승격]` 클릭 시 확인 모달 | [취소] / [승격] |
| 삭제/대량 변경 | 처리 전 사전 확인 모달 | 승인/거부 |

### 취소

`post_id` 단위 rollback. 해당 post 의 sentences·`node_sentence_mentions`·`node_category_mentions`·`aliases`·post 행을 CASCADE 로 일괄 정리하고, 공출현 참조가 남지 않은 노드는 고아 정리.

### 인출 과정 사용자 관여

BFS 탐색의 중간 루프를 사용자가 볼 수 있다. 잘못된 방향으로 탐색이 진행되면 사용자가 개입하여 방향 수정 가능 (위 §BFS 탐색 중 시각화 의 `[이 방향 제외하기]`).

## /hypergraph — 전체 누적 그래프 (별도 라우트, 공유 `GraphView` 코어)

조회 전용. 모든 post 의 모든 노드/하이퍼엣지를 통합 시각화. dogfood 검증 도구 — 누적 지식의 허브·도메인 부상 확인, 통찰 승격 후 허브 변화 검증.

이 라우트가 **공유 `GraphView` 코어 컴포넌트**의 표준 사용처. F7d (`/note` 그래프 패널) 와 F9 (`/synapse` 그래프 패널) 는 이 코어를 데이터 어댑터 (`postId` / `nodeIds` 필터) + 작은 wrapper 로 재사용.

### 시각화 가치 (왜 보는가)

| 가치 | 시각 강조 |
|---|---|
| A. **허브 부상** (원칙 10) | 노드 크기 = `mention degree` 비례. 허브가 즉시 또렷이 큼 |
| B. **통찰 중력** (원칙 15-2) | `origin='insight'` 노드 = `--accent-2` + 가장 큼 + 외곽 링 + 강한 글로우 |
| C. **고립 발견** | 고립 노드 = 작은 점 (2.5px) + `--text-4`, 외곽 영역에 배치 |
| D. **도메인 분포** | 노드 색상 = 시드 카테고리 7 색 (BOD/MND/FOD/WRK/REL/LIV/TIM — 일상 대분류 7) |

이 4 가치 위에서 **시각의 주인공은 노드** — 바구니 (sentence·category) 는 클릭 시 우측 detail panel 에 텍스트로 풀림 (옵션 ε, node-centric). 바구니를 별도 노드로 그리거나 (β) 반투명 영역으로 그리면 (γ) 큰 그래프 (937+ 노드) 에서 시각 잼 + 허브 부상이 묻혀 채택하지 않음.

### 동작 사양

- **시각화 엔진**: WebView + **vis-network** (큰 그래프 검증된 표준, force-layout / 줌·팬 / 클러스터링 자동). Flutter `GraphView` 추상 인터페이스 + `VisNetworkGraphView` 구현 — 추후 native (`CustomPainter` + Barnes-Hut) 구현 추가 여지
- **노드 색상**: `CAT_COLORS[node.primaryCategoryCode]` (시드 19 중 일상 대분류 7 — BOD/MND/FOD/WRK/REL/LIV/TIM), 통찰 = `--accent-2`, 고립 = `--text-4`. 시드 19 의 다른 12 코드 (PER/MON/TEC/EDU/LAW/TRV/NAT/CUL/HOB/SOC/REG/ACT) 는 `--node` (회색) 로 fallback
- **노드 반지름**: `min(14, 3 + sqrt(degree) * 2.2)` (4~14px), insight=16px, isolated=2.5px
- **노드 글로우**: hub `0 0 8px {fill}88`, insight `0 0 14px rgba(217,188,131,0.7)` + 외곽 링 (`--accent-line`, r+6)
- **하이퍼엣지 시각화 (멤버십 선)**: 노드와 노드 사이 선 = 같은 바구니 공출현 횟수. 굵기 `min(3.5, 0.5 + weight * 0.4)` (0.5~3.5px). 색상 기본 `--border-strong`, 클릭 노드 인접 = `--accent-line`
- **라벨**: JetBrains Mono 9~12px, 노드 위 textAnchor middle. insight 12px·600, hub 11px·600 + ` · {degree}` 부가
- **노드 클릭** (hover 갈음 — 모바일·데스크톱 통일):
  - 디밍: 클릭 노드 + 인접 외 노드 opacity 0.25 / 엣지 0.12
  - 우측 detail panel (모바일 하단 시트) 채움: 같은 sentence 바구니 멤버 + 원문 (시간순 N개) / 같은 카테고리 / 별칭 / 통계 / [▷ 이 노드 보유 노트로 이동]
  - 빈 캔버스 클릭 → 디밍 해제 + detail panel 빈 안내
  - 같은 노드 두 번째 클릭 → 디밍 해제
- **검색**: 노드 이름·별칭 substring
- **필터**: 카테고리 7 체크박스 (BOD/MND/FOD/WRK/REL/LIV/TIM — 색상 dot 표기) + `[기타 12]` 토글 (시드 19 중 나머지 코드를 토글로 펼침), 칩 (전체 / 허브 / `✦` 통찰 / 고립만), BFS 깊이 슬라이더 (0~5)
- **Legend (좌상단 floating card)**: 크기 = degree / 두께 = 가중치 / `✦` = 통찰
- **줌 컨트롤 (우하단)**: `[+] [fit] [-]`
- **수정·삭제는 `/note` 또는 `/review` 로 이동** — 조회 뷰는 read-only

### F7d / F9 재사용 (`GraphView` 코어 어댑터)

같은 `GraphView` 코어 컴포넌트를 두 wrapper 가 재사용:

| Wrapper | 데이터 어댑터 | UI 차이 |
|---|---|---|
| `/note` 그래프 패널 (F7d) | `WHERE post_id = ?` 필터 | 우측 280px 패널 + ⊟ 접기 / 모바일 `_showGraph` 토글. 좌측 검색·필터·깊이 슬라이더 숨김. 우측 detail panel 통합 (별도 컬럼 X) |
| `/synapse` 그래프 패널 (F9) | retrieve 캐시 `nodeIds` 필터 | 우측 280px 패널 + ⊟ 접기. retrieve 캐시 카운트 + `✦ 통찰 N` 통계만 |
