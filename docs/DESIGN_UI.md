# Synapse 설계 — UI/UX

UI 동작 원칙은 **`docs/DESIGN_PRINCIPLES.md §5`** 참고.

## 라우트 (모바일 우선·데스크톱 통합)

같은 Flutter 위젯이 모바일·데스크톱에서 자연 확장. post 사이드바는 모바일에선 슬라이드 패널(`☰`), 데스크톱에선 고정 패널로 노출.

- `/note` — 지식 축적 (자동저장 + 의미 처리)
- `/synapse` — 인출·융합 세션 (통찰 승격)

`/hypergraph` (조회 전용), `/review` (검수 전용) 는 후속 PLAN.

## /note — 지식 축적

### 데스크톱

```
┌──────────────────────────────────────────────────────────────────────┐
│ SYNAPSE                                       /note      /synapse    │ ← 상단 바
├───────────────────────┬──────────────────────────────────────────────┤
│ ＋ 새 노트             │  ✓ 저장됨 (방금)                ⌘S 정리     │ ← 자동저장 상태 + 트리거
│                       │  ─────────────────────────────────────────── │
│ ── 노트 ──            │                                              │
│ 12  허리 치료 경과    │  # 더나은                                    │
│      4분 전           │  ## 개발팀                                   │
│ ─────────────         │                                              │
│ 11  팀 회의록         │  - 팀장:: 박지수                             │
│      2시간 전         │  - 프론트엔드 김민수                          │
│ ─────────────         │                                              │
│ 10  ✦ 시냅스 위에서   │  오늘 회의가 세 개 연속이었다.                 │
│     모바일 우선 결정   │  김민수가 새 컴포넌트 시안 잡아왔다.           │
│      어제             │                                              │
│                       │                                              │
│ ── 시냅스 ──          │                                              │
│ 9  허리 치료 진행     │                                              │
│     3시간 전          │                                              │
│                       │                                              │
└───────────────────────┴──────────────────────────────────────────────┘
```

### 모바일

```
┌──────────────────────────┐
┌──────────────────────────┐
│ ☰  SYNAPSE          /◐  │ ← 햄버거 + 라우트 토글
├──────────────────────────┤
│ ✓ 저장됨        ⌘S 정리 │
│ ──────────────────────── │
│                          │
│ # 더나은                 │
│ ## 개발팀                │
│                          │
│ - 팀장:: 박지수          │
│ - 프론트엔드 김민수       │
│                          │
│ 오늘 회의가 세 개 연속이  │
│ 었다.                    │
│ 김민수가 새 컴포넌트      │
│ 시안 잡아왔다.            │
│                          │
└──────────────────────────┘
```

`☰` 탭 → post 사이드바 슬라이드.

### post 사이드바

- 번호 + 제목 + 수정시간
- `kind='insight'` post 는 `✦` 아이콘 + 앰버 보더
- `kind='note'` / `kind='synapse'` 섹션으로 그룹화
- 호버 시 [삭제] 액션 (확인 모달 후 sentences·mentions CASCADE)
- 클릭 시 source 그대로 로드 (재진입·이어쓰기)

### 자동저장 상태 + 의미 처리 트리거

상단 바에 짧은 상태 줄로 노출. 두 동작이 **시각적으로 구분**:

| 상태 | 표시 | 트리거 |
|---|---|---|
| 자동저장 | `변경됨` → `저장 중…` → `✓ 저장됨 (방금)` | 1.5초 디바운스 + 페이지 이탈 |
| 의미 처리 | `⌘S 정리` 버튼 → `⚙ 정리 중…` → `✦ 정리됨` | 사용자 명시 (⌘S 또는 버튼) |

자동저장은 항상 백그라운드, 의미 처리는 사용자가 의도를 줄 때만.

### LLM 정정 카드 인라인

의미 처리 직후 / 재진입 시 본문 끝에 카드로 노출. 자동 적용 금지.

```
─── LLM 정정 후보 ───
┌──────────────────────────────────────┐
│  스ㅏ타벅스 → 스타벅스              │
│  근거: 자모 거리 1, 별칭 미등록       │
│  [적용]  [무시]                      │
└──────────────────────────────────────┘
┌──────────────────────────────────────┐
│  김민수 → 김민슈                     │
│  근거: 자모 거리 1                    │
│  [적용]  [무시]                      │
└──────────────────────────────────────┘
```

- 별칭 등록 토큰 (`스벅` 등) 은 카드에 안 뜸 (사용자 정감 보존).
- [적용] 클릭 시: aliases INSERT (origin='user') + posts.source / sentences 갱신.
- [무시] 클릭 시: 카드만 닫힘. 같은 토큰 캐싱되어 다음 의미 처리 때 다시 안 뜸.

### 데스크톱

```
┌──────────────────────────────────────────────────────────────────────┐
│ SYNAPSE                                       /note      /synapse    │
├───────────────────────┬──────────────────────────────────────────────┤
│ ＋ 새 시냅스           │                                              │
│                       │  ╭─────────────────────────────────────╮    │
│ ── 시냅스 ──          │  │ Q: 허리 어떻게 진행되고 있지?         │    │

│     ◯ 활성            │                                              │
```

### BFS 탐색 중 시각화

답변 생성 전 단계가 사용자에게 보임. 잘못된 방향으로 탐색이 진행되면 사용자 개입 가능.

```
◌ 탐색 중...

  허리 ●─── 디스크 ●─── L4-L5 ●
       ╲                       ╲
        병원 ○                  진단 ○
        ╲                       ╲
         강남세브란스 ○          물리치료 ○

  ● 진입한 노드   ○ 탐색 예정 노드
  [이 방향 제외하기]
```

복잡해지면 허브 노드 2~3개만 표시하고 "...외 N개 노드" 처리.

### 답변 카드 + 통찰 승격

```
╭──────────────────────────────────────────╮
│ A: ...답변 본문 (마크다운 렌더)...        │
│                                          │
│  [⬆ 통찰로 승격]   [재질문]   [복사]     │
╰──────────────────────────────────────────╯
```

`[⬆ 통찰로 승격]` 클릭 → 확인 모달:

```
┌──────────────────────────────────────┐
│  통찰로 승격하시겠어요?                │
│                                      │
│  새 ✦ 통찰 post 가 생성되며,           │
│  이 시냅스 세션에서 함께 발화한        │
│  노드 12 개가 자동 연결됩니다.         │
│                                      │
│  [취소]                  [승격]      │
└──────────────────────────────────────┘
```

승격 후:
- `/note` 사이드바에 `✦` 아이콘 + 앰버 보더로 새 post 표시
- 본체 sentence 는 편집 불가 (시도 시 토스트 "통찰은 편집할 수 없어요. 삭제는 /review 에서.")
- 다음 retrieve 에서 우선 점화 (다음 시냅스의 중력)

## /hypergraph — 후속 PLAN

조회 전용 시각화. vis-network 기반. 누적 노드가 사용자 인지 한계 (~200 노드) 도달 시 별도 PLAN 으로 작성.

핵심 동작:
- 허브 노드 중심 시각화 (앰버 채움)
- 하이퍼엣지(문장·카테고리 바구니)는 노드들을 감싸는 반투명 영역 또는 여러 노드를 동시에 잇는 선
- 노드 탭 → 같은 바구니 멤버 + 원문 sentence 확인
- `origin='insight'` sentence 는 허브 크기 + 와 앰버 링으로 강조
- 수정·삭제는 `/note` 또는 `/review` 로 이동 (조회 뷰는 read-only)
