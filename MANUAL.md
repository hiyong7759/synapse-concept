# Synapse 매뉴얼

## 이게 뭔가요?

당신이 LLM(ChatGPT, Claude 등)에게 질문할 때,
LLM은 **당신이 누군지 모릅니다.**

"운동 추천해줘"라고 하면 일반적인 운동을 알려줍니다.
당신에게 허리디스크가 있어도요.

"빌드가 안 돼"라고 하면 "어떤 OS 쓰세요?"부터 물어봅니다.
매번요.

Synapse는 이 문제를 해결합니다.
**당신의 정보를 기억해두고, 질문할 때 자동으로 맥락을 넣어줍니다.**

---

## 어떻게 동작하나요?

### 1. 당신의 정보를 그래프로 저장합니다

```
맥미니M4 ──(OS)── macOS
맥미니M4 ──(IDE)── VS Code
맥미니M4 ──(위치)── 집

허리디스크 ──(부위)── L4-L5
허리디스크 ──(주의운동)── 데드리프트
데드리프트 ──(대안)── 트랩바
```

- **노드** = 단어 하나. "맥미니M4", "Docker", "허리디스크"
- **엣지** = 단어 사이의 관계. "맥미니M4의 OS는 macOS"
- 트리가 아닙니다. 모든 노드는 동등하고, 엣지로 연결될 뿐입니다.

### 2. 질문하면 관련된 것만 꺼냅니다

```
질문: "맥미니 개발환경"
  ↓
키워드 "맥미니", "개발환경" → 두 노드 매칭
  ↓
둘이 연결됨 → 교집합 (맥미니 ∩ 개발환경)
  ↓
결과: macOS, VS Code, Docker, Node.js, Git, Homebrew
  ↓
LLM에게 전달 → 맥미니 기준 개발환경 답변
```

### 3. 대화하면서 자연스럽게 쌓입니다

특별한 입력 화면이 없습니다.
대화하다가 새로운 정보가 나오면 "시냅스에 저장해"라고 하면 됩니다.

```
당신: "요즘 RTX3080으로 llama.cpp 돌려보고 있어"
AI: 📝 다음 내용을 시냅스에 저장할까요?

노드:
  - RTX3080 (장비)
  - llama.cpp (기술)

관계:
  - RTX3080 ──(용도)── llama.cpp

[Y/n]
```

## 처음 시작할 때 (온보딩)

그래프가 비어있으면 맥락도 없습니다.
처음에 당신에 대해 조금만 알려주면 바로 달라집니다.

**편한 것부터, 편한 만큼만:**

| 알려줄 수 있는 것 | 이게 왜 좋은가 |
|------------------|---------------|
| 이력서/자기소개 | 기술, 경력, 프로젝트를 한 번에 구축 |
| GitHub URL | 실제 사용 기술 파악 |
| 개발 환경 | 환경 맥락 |
| 진행 중인 프로젝트 | 작업 맥락 |
| 건강/제약 | 항상 고려되는 안전 정보 |
| 관심사 | 일상 질문의 맥락 |

하나만 알려줘도 시작됩니다.
**15개 노드, 4개 분야**가 쌓이면 질문의 75%에서 맥락이 동작합니다.

## 핵심 개념

### 1. 노드는 원자입니다

"React Native"는 그래프에 딱 하나만 존재합니다.
같은 개념을 두 번 만들지 않습니다.

다른 이름으로도 찾을 수 있게 **별칭(aliases)**을 등록합니다:

```
aliases:
  "이력" → 이력서 노드
  "업무분석가" → BA 노드
  "프로젝트매니저" → PM 노드
```

### 2. 엣지는 선택자입니다

노드가 많아져도 괜찮습니다.
질문 키워드에 매칭된 노드에서 **정방향 엣지를 따라** 연결된 것만 가져옵니다.

연결된 시작 노드끼리는 **교집합**으로 범위를 좁힙니다.
"맥미니 개발환경" → 맥미니에도 있고 개발환경에도 있는 것만.

역방향이 필요하면 **양방향 엣지를 추가**하면 됩니다.
탐색 로직을 복잡하게 만들지 않고, 데이터를 정확하게 넣는 게 답입니다.

### 3. safety 노드는 항상 따라옵니다

허리디스크, 알레르기 같은 정보는 **safety** 표시를 합니다.
관련 질문에서 항상 맥락에 포함됩니다.

```json
{"name": "허리디스크", "safety": true, "safety_rule": "운동 관련 질문 시 반드시 고려"}
```

## 틀렸을 때 고치는 법

모든 답변 끝에 **어떤 맥락이 쓰였는지** 표시됩니다:

```
📌 맥락: 맥미니M4, macOS, Docker
```

이걸 보고:

- **"이거 아니야"** → 잘못된 노드 수정/삭제
- **"이것도 넣어줘"** → 빠진 관계 추가
- **"이 연결 틀렸어"** → 엣지 수정

매칭 실패 시 missing 안내가 나옵니다:
```
💡 "건강" 관련 노드가 없습니다. 추가하시겠어요?
```

## 개인정보는 안전합니다

- 모든 데이터는 **당신의 기기에만** 저장됩니다 (SQLite 파일 하나)
- 서버에 올라가는 것은 없습니다
- 암호화해서도 서버에 저장하지 않습니다 (의도적 설계)

## 사용 가능한 명령어

| 하고 싶은 것 | 말하면 되는 것 |
|-------------|--------------|
| 맥락 자동 반영 | 그냥 질문하세요. 자동입니다. |
| 정보 저장 | "시냅스에 저장해", "기억해" |
| 저장된 것 보기 | "내 메모리 보여줘", "시냅스 목록" |
| 수정/비활성화 | "이거 수정해", "비활성화해" |
| 복원 | "복원해줘" |

---

## 정리

1. **대화하면서 자연스럽게 쌓이고**
2. **질문하면 자동으로 맥락이 들어가고**
3. **틀리면 고치면 됩니다**

당신이 쌓을수록, 당신에게 맞는 답이 나옵니다.
