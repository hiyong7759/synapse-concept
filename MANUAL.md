# Synapse 매뉴얼

## 이게 뭔가요?

당신이 LLM(ChatGPT, Claude 등)에게 질문할 때,
LLM은 **당신이 누군지 모릅니다.**

"운동 추천해줘"라고 하면 일반적인 운동을 알려줍니다.
당신에게 허리디스크가 있어도요.

"빌드가 안 돼"라고 하면 "어떤 OS 쓰세요?"부터 물어봅니다.
매번요.

Synapse는 이 문제를 해결합니다.
**당신의 정보를 기억해두고, 질문할 때 자동으로 맥락을 넣어줍니다.**

---

## 어떻게 동작하나요?

### 1. 당신의 정보를 그래프로 저장합니다

```
맥미니 ──(스펙)── M4
맥미니 ──(OS)── macOS
맥미니 ──(설치됨)── Docker

허리디스크 ──(부위)── L4-L5
허리디스크 ──(주의운동)── 데드리프트
데드리프트 ──(대안)── 트랩바
```

- **노드** = 단어 하나. "맥미니", "Docker", "허리디스크"
- **엣지** = 단어 사이의 관계. "맥미니의 OS는 macOS"
- 트리가 아닙니다. 모든 노드는 동등하고, 엣지로 연결될 뿐입니다.

### 2. 질문하면 관련된 것만 꺼냅니다

```
질문: "운동 추천해줘"
  ↓
키워드 "운동" → 데드리프트, 트랩바 매칭
  ↓
연결된 노드 탐색: 허리디스크, L4-L5, 재발, 허리부담적음
  ↓
safety 노드 확인: 허리디스크 → "운동 관련 질문 시 반드시 고려"
  ↓
LLM에게 전달: "이 사람은 L4-L5 허리디스크가 있고,
               데드리프트는 재발 위험이 있으며,
               트랩바가 대안입니다"
```

LLM은 이 맥락을 받아서 **당신에게 맞는 답**을 합니다.

### 3. 대화하면서 자연스럽게 쌓입니다

특별한 입력 화면이 없습니다.
대화하다가 새로운 정보가 나오면 "시냅스에 저장해"라고 하면 됩니다.

```
당신: "요즘 RTX3080으로 llama.cpp 돌려보고 있어"
AI: 📝 다음 내용을 시냅스에 저장할까요?

노드:
  - RTX3080 (장비)
  - llama.cpp (기술)

관계:
  - RTX3080 ──(GPU)── llama.cpp

[Y/n]
```

확인하면 저장. 다음부터 "로컬 LLM 추천해줘"라고 하면
RTX3080과 llama.cpp 경험이 맥락에 포함됩니다.

## 처음 시작할 때 (온보딩)

그래프가 비어있으면 맥락도 없습니다.
처음에 당신에 대해 조금만 알려주면 바로 달라집니다.

**편한 것부터, 편한 만큼만:**

| 알려줄 수 있는 것 | 예시 | 이게 왜 좋은가 |
|------------------|------|---------------|
| 이력서/자기소개 | PDF, 텍스트, 그냥 말로 | 기술, 경력, 프로젝트를 한 번에 구축 |
| GitHub URL | github.com/username | 실제 사용 기술 파악 |
| 개발 환경 | "맥미니 M4, Docker 사용" | 환경 맥락 |
| 진행 중인 프로젝트 | "React Native로 앱 만드는 중" | 작업 맥락 |
| 건강/제약 | "허리디스크", "땅콩 알레르기" | 항상 고려되는 안전 정보 |
| 관심사 | "홈트레이닝", "로컬 LLM" | 일상 질문의 맥락 |

하나만 알려줘도 시작됩니다.
**15개 노드, 4개 분야**가 쌓이면 질문의 75%에서 맥락이 동작합니다.

---

## 핵심 개념 3가지

### 1. 노드는 원자입니다

"React Native"는 그래프에 딱 하나만 존재합니다.
같은 개념을 두 번 만들지 않습니다.

대신 **부르는 이름이 여러 개**일 수 있습니다:

```
React Native ──(same)── 리액트 네이티브
이력 ──(same)── 이력서
이력 ──(same)── 프로필
RTX3080 ──(same)── 3080
Kubernetes ──(same)── k8s
```

영어든 한글이든 약칭이든, 어떤 이름으로 검색해도 같은 노드에 도달합니다.
이 연결은 저장할 때 자동으로 만들어집니다.

### 2. 엣지는 선택자입니다

노드가 많아지는 건 괜찮습니다.
질문할 때 키워드에 매칭된 노드에서 출발해서,
**엣지를 따라 연결된 것만** 가져옵니다.

노드 1만 개가 있어도 "운동 추천해줘"에는
운동 관련 노드만 나옵니다.

가져온 결과가 좀 넓어도 괜찮습니다.
LLM이 그중에서 필요한 것만 골라 쓰니까요.
당신이 몇십 번 대화에 걸쳐 쌓은 맥락인데,
넉넉하게 쓰이는 게 맞습니다.

### 3. safety 노드는 항상 따라옵니다

허리디스크, 알레르기 같은 정보는 **safety** 표시를 합니다.

```json
{"name": "허리디스크", "safety": true, "safety_rule": "운동 관련 질문 시 반드시 고려"}
{"name": "땅콩알레르기", "safety": true, "safety_rule": "음식 관련 질문 시 반드시 고려"}
```

safety 노드는 관련 질문에서 항상 맥락에 포함됩니다.
"운동 추천해줘"라고 했을 때 허리디스크를 빼먹지 않습니다.

## 틀렸을 때 고치는 법

Synapse는 처음부터 완벽할 필요 없습니다.

모든 답변 끝에 **어떤 맥락이 쓰였는지** 표시됩니다:

```
📌 맥락: 맥미니, M4, Docker, macOS
```

이걸 보고:

- **"이거 아니야"** → 잘못된 노드 수정/삭제
- **"이것도 넣어줘"** → 빠진 관계 추가
- **"이 연결 틀렸어"** → 엣지 수정

```
"내 메모리 보여줘"     → 저장된 노드/엣지 목록
"React Native 삭제해"  → 노드 비활성화
"맥미니 도메인 수정해"  → 속성 변경
```

고칠수록 그래프가 정확해지고, 맥락이 정확해지고, 답변이 정확해집니다.

---

## 개인정보는 안전합니다

- 모든 데이터는 **당신의 기기에만** 저장됩니다 (SQLite 파일 하나)
- 서버에 올라가는 것은 없습니다
- LLM에게 전달될 때 **어떤 노드가 노출되었는지** 기록됩니다 (exposure log)
- LLM 서비스를 바꿀 때, 이전에 노출된 적 없는 정보는 새 동의가 필요합니다

## 사용 가능한 명령어

| 하고 싶은 것 | 말하면 되는 것 |
|-------------|--------------|
| 맥락 자동 반영 | 그냥 질문하세요. 자동입니다. |
| 정보 저장 | "시냅스에 저장해", "기억해", "이거 기록해둬" |
| 저장된 것 보기 | "내 메모리 보여줘", "시냅스 목록" |
| 수정/삭제 | "이거 수정해", "삭제해", "도메인 바꿔줘" |
| 처음 설정 | "시냅스 시작", "온보딩" |

---

## 용량은 걱정 안 해도 됩니다

| 사용 패턴 | 예상 용량 |
|----------|----------|
| 1년 일반 사용 | ~3MB |
| 5년 헤비 유저 | ~30MB |
| LLM 하루 100회, 10년 | ~2GB |

사진 몇 장 수준입니다.
안 쓰는 노드는 삭제하면 되고, 삭제해도 복원할 수 있습니다.

---

## 정리

1. **대화하면서 자연스럽게 쌓이고**
2. **질문하면 자동으로 맥락이 들어가고**
3. **틀리면 고치면 됩니다**

당신이 쌓을수록, 당신에게 맞는 답이 나옵니다.
