# Synapse Context — 맥락 프롬프트 생성

사용자의 질문에서 키워드를 추출하고, 개인 지식 그래프에서 관련 맥락을 가져와 프롬프트에 포함한다.

## 실행 절차

### 0단계: 온보딩 상태 확인

맥락 생성 전에 먼저 그래프가 충분한지 확인한다:

```bash
python3 scripts/check_onboarding.py
```

결과의 `status` 값에 따라 분기:
- `"complete"` → 1단계로 진행 (정상 맥락 생성)
- `"onboarding"` → 온보딩 흐름 실행 (아래 "온보딩" 섹션)

### 1단계: 맥락 생성

1. 사용자 질문에서 핵심 키워드를 추출한다.
2. `scripts/get_context.py`를 실행하여 서브그래프를 추출한다:
   ```bash
   python3 scripts/get_context.py "<사용자 질문>"
   ```
3. 결과의 `prompt` 필드를 시스템 맥락으로 사용한다.
4. 반환된 `nodes_used`를 확인하고, 맥락 판단을 수행한다:
   - 질문과 무관한 노드가 포함되어 있으면 제외한다.
   - 질문 맥락에서 유효한 노드만 최종 프롬프트에 포함한다.
5. `safety_nodes`는 질문과 관련될 때만 고려한다.
   - 운동/신체 질문 → 허리디스크 safety 적용
   - 음식 질문 → 알레르기 safety 적용
   - 날씨 질문 → safety 무시
6. 최종 맥락을 답변에 반영한다.

## 맥락 판단 기준

- 질문의 의도와 직접적으로 관련된 노드만 포함
- 예: "운동 추천해줘" → 데드리프트 ✓, 트랩바 ✓, 시디즈T80(의자) ✗
- 예: "빌드가 안 돼" → macOS ✓, Expo ✓, 집 ✗

## 출력 형식

### 맥락이 있을 때

답변 끝에 사용된 노드 목록을 간단히 표시:
```
📌 맥락: 맥미니, M4, Docker, macOS
```

### 맥락이 없을 때 — 피드백

온보딩이 완료된 상태인데 매칭이 0개이면, 사용자에게 알려준다.
이걸 숨기면 사용자는 맥락이 빠진 줄도 모르고, 그래프를 개선할 기회도 없다.

```
💡 이 질문에 매칭되는 맥락이 없었습니다.
   "이력서"와 관련된 노드를 추가하면 다음부터 맥락이 반영됩니다.
   예: "시냅스에 저장해 — 이력서 관련 기술스택이랑 프로젝트"
```

핵심은:
- **왜 안 됐는지** 키워드를 보여줌 (검색된 키워드: "이력서")
- **어떻게 고치는지** 구체적 행동을 제안 (저장 명령 예시)
- 한 줄로 짧게. 매번 길게 설명하면 짜증남.

## 온보딩

`check_onboarding.py` 결과가 `"onboarding"`이면 이 흐름을 실행한다.

### 완료 기준

시뮬레이션으로 검증된 최소 기준:

단계별 시뮬레이션(find_threshold.py)으로 검증된 기준:

| 항목 | 기준 | 근거 |
|------|------|------|
| 총 노드 | 15개 이상 | 15노드 시점에서 75% 개별화 달성 |
| 도메인 수 | 4개 이상 | 새 도메인 추가가 개별화율 점프의 핵심 요인 |
| 커버된 도메인 | 3개 이상(각 2+노드) | 단일 노드 도메인은 맥락 품질 낮음 |

핵심 발견: 노드 수보다 도메인 다양성이 중요하다.
같은 도메인에 노드 4개를 추가해도 개별화율은 변하지 않았지만,
새 도메인 1개를 추가하면 +17%p 점프했다.

### 첫 방문 안내

```
시냅스는 당신의 경험과 맥락을 기억해서,
어떤 질문을 하든 "당신에게 맞는 답"을 만들어주는 지식 그래프입니다.

처음에 당신에 대해 조금만 알려주시면,
그때부터 모든 대화가 달라집니다.

아래 중 편한 것부터 알려주세요. 전부 안 해도 됩니다.
대화하면서 자연스럽게 쌓을 수도 있어요.

1. 📄 이력서나 자기소개 (파일, 텍스트, 링크 뭐든)
2. 🔗 GitHub URL
3. 💻 현재 개발 환경 (장비, OS, 주요 도구)
4. 🚀 지금 하고 있는 프로젝트
5. ⚠️ 건강이나 알레르기 같은 주의사항
6. 🎯 관심사나 취미

하나만 알려줘도 시작할 수 있습니다.
많이 알려줄수록 맥락이 풍부해집니다.
모든 정보는 로컬에만 저장됩니다.
```

### 정보를 받으면

사용자가 정보를 주면:

- **이력서/텍스트**: 내용을 분석하여 노드/엣지 추출 → save 스킬 흐름으로 확인 후 저장
- **GitHub URL**: 프로필, 주요 레포의 언어/프레임워크/프로젝트명 추출 → 노드화
- **구두 설명**: 대화에서 키워드 추출 → 노드/엣지화
- **건강/제약**: safety 노드로 등록 (safety_rule 포함)

정보를 저장할 때마다 `check_onboarding.py`를 다시 실행하여 진행 상황을 확인한다.

### 진행 중 안내

온보딩이 아직 미완이면 `suggestions` 필드의 내용을 자연스럽게 안내한다:

```
좋아요! [N]개 노드가 저장되었습니다.

지금 상태:
  ✅ 기술 (8개) — 개발 관련 질문에 맥락 반영 가능
  ✅ 장비 (3개) — 환경 관련 질문에 맥락 반영 가능
  ⬜ 건강/운동 — 아직 없어요
  ⬜ 관심사/취미 — 아직 없어요

더 알려주실 게 있으면 계속하시고, 나중에 대화하면서 채워도 됩니다.
```

### 완료 안내

모든 기준을 충족하면:

```
시냅스 준비 완료! [N]개 노드, [M]개 관계가 저장되었습니다.
이제부터 질문하시면 이 맥락이 자동으로 반영됩니다.

한번 시도해보세요 — 아무 질문이나 해보시면 차이를 느끼실 겁니다.
```

## 에러 처리

- DB가 없으면: `python3 scripts/init_db.py` 로 자동 초기화
- 스크립트 실행 실패 시: 일반 답변으로 폴백
