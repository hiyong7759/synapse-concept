# Synapse Save — 대화에서 지식 추출 및 저장

현재 대화에서 개념(노드)과 관계(엣지)를 추출하여 개인 지식 그래프에 저장한다.

## 실행 절차

### 1단계: 대화 분석 및 추출

현재 대화를 분석하여 다음을 추출한다:

**노드 (개념/단어):**
- 기술: React Native, Docker, SQLite 등
- 장비: 맥미니, M4 등
- 프로젝트: Poomacy, Synapse 등
- 건강: 허리디스크, L4-L5 등
- 판단: 배제한 선택지, 결정 사항 등
- 기타 구체적 개념

**엣지 (관계):**
- link: 두 개념의 관련성 + label (위치, 스펙, 용도, 원인 등)
- similar: 의미가 비슷한 것
- same: 같은 것의 다른 표현

**추출 규칙:**
- 모든 노드는 수평하다. 상하 관계 없음.
- 구체적인 단어/개념만 노드로. 문장이 아님.
- 엣지 label은 관계의 성격을 간결하게 표현.
- AI가 설명한 것과 사용자가 직접 말한 것을 구분할 필요 없음 (1단계).
- 실패 경험, 배제한 선택지도 포함.

**노드 원자성:**
노드는 원자다. 같은 개념이면 반드시 기존 노드를 재사용한다.
중복 노드가 생기면 그래프가 오염되고, 같은 개념이 다른 경로로 분산되어 맥락 품질이 떨어진다.

```
"React Native"가 이미 있으면 → 기존 노드에 엣지만 추가
"react native" (대소문자 다름) → 같은 노드로 인식
절대 같은 개념을 두 번 만들지 않는다.
```

엣지는 선택자다. 노드 사이를 잇는 경로이므로 많아져도 문제없다.
BFS가 엣지를 따라 탐색하고, LLM이 결과를 필터링하는 구조이기 때문에
엣지가 늘어나는 것은 맥락이 풍부해지는 것이지 부하가 아니다.

**동의어/유사어 자동 연결:**
노드를 추출할 때, 자연스러운 동의어·유사어·약칭을 same/similar 엣지로 함께 생성한다.
사용자가 일일이 알려줄 필요 없이, LLM이 판단하여 자동으로 연결한다.

```
동의어:
  이력서 ──(same)── 이력
  이력서 ──(same)── 프로필

약칭:
  RTX3080 ──(same)── 3080

다국어 (같은 개념의 다른 문자 표기):
  React Native ──(same)── 리액트 네이티브
  Docker ──(same)── 도커
  Kubernetes ──(same)── 쿠버네티스
  Kubernetes ──(same)── k8s
```

이 연결이 있어야 사용자가 어떤 언어, 어떤 표현을 써도 같은 맥락이 나온다.
노드는 하나, 부르는 이름이 여러 개인 것.

### 2단계: 사용자 확인

추출 결과를 사용자에게 보여주고 확인받는다:

```
📝 다음 내용을 시냅스에 저장할까요?

노드:
  - React Native (기술)
  - Expo (기술)
  - Poomacy (프로젝트)

관계:
  - React Native ──(프레임워크)── Poomacy
  - Expo ──(빌드도구)── React Native

[Y/n] 또는 수정할 내용을 알려주세요.
```

사용자가 수정 요청하면 반영. 확인하면 저장 진행.

### 3단계: 저장 실행

확인된 데이터를 JSON으로 구성하여 실행:

```bash
python3 scripts/add_nodes.py '<json>'
```

JSON 형식:
```json
{
  "nodes": [
    {"name": "React Native", "domain": "기술"},
    {"name": "Expo", "domain": "기술"},
    {"name": "Poomacy", "domain": "프로젝트"}
  ],
  "edges": [
    {"source": "React Native", "target": "Poomacy", "type": "link", "label": "프레임워크"},
    {"source": "Expo", "target": "React Native", "type": "link", "label": "빌드도구"}
  ]
}
```

### safety 노드 설정

건강, 알레르기 등 항상 포함되어야 하는 정보는 safety로 마크:
```json
{"name": "허리디스크", "domain": "건강", "safety": true, "safety_rule": "운동 관련 질문 시 반드시 고려"}
```

## 에러 처리

- DB가 없으면 자동 초기화
- 이미 존재하는 노드는 중복 생성하지 않음 (기존 ID 재사용)
